R3 — Segregation of duties

  • The agent who built a card can never be the one who verifies it
  • Enforced in code — not by convention
  • Litmus is the designated independent verifier for all cards
  • Prevents self-ratification; preserves integrity of the "done" state

Control layer 1 — Authority rails

  • Rules R1–R9, E1, and a stage-gate guard every write to the ledger
  • Fail-closed: if a rule cannot be evaluated, the write is denied
  • Implemented in code — not documented conventions
  • Example: Northstar physically cannot author backlog cards
  • Reserved work cannot auto-complete regardless of autonomy stage
  • Every rule violation is itself a ledger event — auditable forever

Control layer 2 — Autonomy stages

  • A single Owner-set dial (0–3) controls how much runs without a human
  • Applies org-wide; individual agent overrides are not possible

Full human approval
1

Currently Stage 2 — an agent may self-complete, but only after an independent verification

Control layer 3 — The harness

  • Lane selection: Path A (subscription) or Path B (metered OpenRouter — default)
  • Bounds: per-run kill switch + weekly spend cap
  • Isolation: sandbox with host isolation, egress allow-list, scoped 1-hour single-repo tokens
  • Safety gates: only an engine that passes all gates may take the next autonomous step
  • The harness is the agent's "hands" — it executes; it also constrains

The invariant

Specialists → Northstar → Ali
  • Agents collaborate peer-to-peer inside working groups
  • Only decisions, major risks, and approvals escalate to the Owner
  • Northstar is the single arbitration and ratification point between specialists and Owner
  • This keeps the Owner informed without burdening them with every detail
